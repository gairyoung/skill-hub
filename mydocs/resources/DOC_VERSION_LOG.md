# Docs 版本日志

> 生成时间：YYYY-MM-DD HH:mm:ss
> 版本：v1.0

## 说明

用于记录 `docs/` 文档版本状态，确保除 `docs/plans/` 外均为最新文档。
本日志只记录关键性更新概要，不追求高频、细粒度流水账。

状态定义：

- `latest`：当前生效版本
- `superseded`：已被替代（需注明替代文件）
- `draft`：草稿，未生效

## 版本记录

| 时间 | 文件路径 | 文档版本 | 状态 | 替代文件 | 变更摘要 |
|------|----------|----------|------|----------|----------|
| YYYY-MM-DD HH:mm:ss | docs/{feature-domain}/arch-spec.md | v1.0 | latest | - | 初始版本 |

## 记录粒度规则（避免过密）

应记录：

1. 文档新增/废弃
2. 架构、流程、数据模型、API 契约变化
3. 验收标准、SLO、发布或回滚策略变化
4. 主/次版本升级

可不记录（仅改文档头部时间）：

- 排版修正、错别字、措辞优化
- 不影响语义的示例补充

## 最新性校验清单

1. 除 `docs/plans/` 外，是否存在同主题多份并存文档。
2. 同主题旧文档是否标记为 `superseded` 并指向替代文档。
3. 本次变更涉及的文档是否都写入了版本记录。
4. 文档头部版本号是否与版本日志一致。
