# Commit Skill

## 版本

- 当前版本：`v1.2.0`

## 概述

该技能用于快速完成一次标准 Git 提交：检查变更、审查风险、暂存文件、自动生成提交信息并执行提交。默认采用引导式选项交互（数字选择）推进流程。

## 交互原则（引导式）

1. 仅使用“选项式提问”，不要求用户手动输入完整命令或自由文本。
2. 每一步给出明确序号（如 `1` / `2` / `3`），用户回复序号即可继续。
3. 合并“提交确认 + push 选择”为一次提问，避免多轮确认。
4. 提交信息由技能基于真实 diff 自动生成，不需要用户确认提交文案。

## 提示文案模板

请选择本次提交策略（回复数字 `1-6`）：

- `1. 提交本次任务相关修改（推荐）`
- `2. 提交本次任务相关修改并推送到远程`
- `3. 提交全部修改`
- `4. 提交全部修改并推送到远程`
- `5. 仅暂存全部修改（不提交）`
- `6. 取消本次提交`

## 适用场景

1. 用户明确要求“提交当前改动”。
2. 目标是一次性提交当前工作区变更。
3. 需要统一采用 Conventional Commits 风格。

## 执行步骤

1. 检查当前状态：
   - `git status`
   - `git diff --staged`
   - `git diff`
2. 快速审查：
   - 是否有明显逻辑错误、安全风险、无关改动。
3. 单次选择（提交 + push）：
   - `1. 提交本次任务相关修改（推荐）`
   - `2. 提交本次任务相关修改并推送到远程`
   - `3. 提交全部修改`
   - `4. 提交全部修改并推送到远程`
   - `5. 仅暂存全部修改（不提交）`
   - `6. 取消本次提交`
4. 根据选择执行：
   - `1`：暂存“本次任务相关修改”（仅本次任务相关文件），自动生成提交信息并 `git commit`。
   - `2`：同 `1`，提交后执行 `git push`。
   - `3`：暂存“全部修改”（工作区全部改动），自动生成提交信息并 `git commit`。
   - `4`：同 `3`，提交后执行 `git push`。
   - `5`：执行 `git add -A`，并结束流程（不提交、不推送）。
   - `6`：结束流程，不做任何提交动作。
5. 提交信息策略（自动）：
   - 使用 Conventional Commits：`<type>(<scope>): <description>`。
   - 由技能基于 staged diff 自动生成，不再单独询问用户确认文案。

## 约束

1. 提交前必须基于真实 diff，而非猜测。
2. 不要把无关文件混入同一次提交。
3. 若存在阻断问题，先反馈问题再提交。
4. 所有交互优先采用数字选项，不要求用户手写复杂输入。
5. 未经用户选择，不执行 `git push`。
